# AGENT.md

This file provides guidance to Clew Code when working with code in this repository.

## Build / Test / Lint

```bash
bun run dev              # Live reload (--watch)
bun run build            # Production build to dist/
bun run start            # Run compiled build
bun test                 # Full Vitest suite
bun test --bail          # Stop on first failure
npx vitest run path/to/file.test.ts   # Single test file
npx vitest run -t "test name"         # Single test by name
bun run check:ci         # Biome CI (lint + format, no fix)
bun x tsc --noEmit       # Type-check only

# Full pre-commit gate:
bun run check:ci && bun x tsc --noEmit && bun test --bail
```

Biome style: 2-space indent, single quotes, 120 cols, LF endings. Scope: `src/**/*.{ts,tsx,js}`.

**Conventional commits required**: `feat:`, `fix:`, `chore:`, `refactor:`, `docs:`, `test:`. Branch naming: `type/description`.
**Always update `CHANGELOG.md`** under `## [Unreleased]` when changing behavior.

## Architecture (non-obvious)

**Entry:** `src/main.tsx` → Ink/React 19 REPL via `src/replLauncher.tsx`. Settings loaded from `.clew/settings.json` + `.clew/settings.local.json`.

**Query loop:** `src/QueryEngine.ts` handles streaming + tool loop. `src/query.ts` is non-streaming. Both route through `src/services/ai/ProviderManager.ts`.

**Tools (76+):** One class per directory under `src/tools/<ToolName>/`, each extending `Tool` from `src/Tool.ts`. Return shape: `{ ok, summary, data? }`. Registered in `src/tools.ts` → `getAllBaseTools()`. Some are feature-gated via `bun:bundle` `--define.*` flags or `process.env`.

**Commands (100+):** One directory per command under `src/commands/<name>/`. Exports `{ name, description, type, handler }`. Registered in `src/commands.ts` → `COMMANDS()` memoized list. Types: `'prompt'` (model-expandable), `'local'` (text output), `'local-jsx'` (Ink UI). Merges built-ins + skills + plugins + MCP skills + dynamic skills.

**Provider system:** `src/services/ai/ProviderManager.ts` unified interface. 29 providers defined in `providers.json`. Per-provider adapters in `adapter/`. Switch mid-session with `/model`.

**Services (40+):** `src/services/` key subsystems:
- `ai/` — provider management (29 adapters, model discovery)
- `mcp/` — MCP client (stdio/SSE/DirectConnect) + OAuth
- `autonomous/` — persistent task queue, cron, retry, dead-letter
- `plugins/` — lifecycle hooks (PreToolUse, PostToolUse, PreBash, PostPrompt)
- `checkpoint/` — structured progress snapshots at 20%/45%/70%
- `goal/` — independent LLM verification of completion

- `compact/` — multi-pass context compression + in-compact memory extraction
- `longTermMemory/` — Dream (7d) + Distill (30d) auto-consolidation
- `lsp/` — Language Server Protocol integration
- `search/` — web search (brave, duckduckgo, serper, tavily)
- `voiceInput/` — browser Web Speech API transcription

**Memory system:** `src/memory/` — SQLite-backed MemoryDB with importance/confidence/access ranking, timeline events, budgeted injection into system prompt. File hierarchy at `.clew/memory/{MEMORY.md,DECISIONS.md,TASTE.md}`. Commands: `/memory init`, `scan`, `rebuild`, `recall`, `feedback`, `search`, `dashboard`.

**Peer / P2P:** `src/peer/PeerServer.ts` + `PeerDiscovery.ts`. UDP multicast + file-based registry. HTTP heartbeat (60s liveness). Message broker with correlation IDs. 15+ peer AI tools + `/peer` commands.

**Skills system:** Two sources — bundled (`src/skills/bundled/`) and project-level (`.clew/skills/*/SKILL.md` Claude Code-compatible). Third extension system alongside tools and commands.

## Key patterns (pain points if missed)

- **ESM only.** Use `node:` prefix for built-ins, `.js` extension for relative imports, `NodeNext` module resolution. `import type` for type-only imports.
- **Never edit `dist/`** — generated by `bun build`. All source in `src/`.
- **Feature flags** are set via `--define.*` in `package.json` scripts. Build defines: `TRANSCRIPT_CLASSIFIER`, `CHICAGO_MCP`, `VOICE_MODE`. Runtime gates: `ENABLE_COMPUTER_USE`, `USER_TYPE`.
- **Gateway mode** replaces Anthropic OAuth. `/login`/`/logout` use `api.clew-code.org`. Key file: `src/utils/gatewayAuth.ts`. `isGatewayConfigured()` returns true by default.
- **Windows:** PowerShell `Set-Clipboard` with UTF-8. Some tools Windows-only (ComputerUse, PowerShellTool). Peer discovery uses both UDP + file registry.
- **Large deps excluded** from bundle via `--external`: electron, playwright, sharp, node-pty, chromium-bidi, onnxruntime, etc.
- **Dashboard** lives in `ClewCode/clew-api` repo (`dashboard/index.html`), NOT in this repo.
- **Settings** are `.clew/settings.json` (shared) + `.clew/settings.local.json` (private). Hooks configured in settings.json.
- **React/Ink UI**: Components in `src/components/`, screens in `src/screens/`, hooks in `src/hooks/`, state in `src/state/`. The terminal UI uses Ink (React for terminals).

## Execution layers (choose the right one)

- **Subagent** (Agent tool) — quick independent read-only investigation
- **Teammate/Swarm** — long-running coordination between named workers
- **LAN Peer** (`/peer`) — another Clew instance on the machine or LAN
- **Process Peer** (`delegate` tool) — local external CLI worker like Codex
- **Background agent** (`/agent`, `/daemon`) — autonomous scheduled execution

For full architecture details see `AGENTS.md`.
