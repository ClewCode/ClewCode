คำสั่ง

Clew มาพร้อม 120+ คำสั่ง slash (411 ไฟล์ใน 120+ ไดเรกทอรี) พร้อมสกิลที่โหลดแบบไดนามิกและคำสั่งปลั๊กอิน คำสั่งมี 3 ประเภท: local (ทำงานในเทอร์มินัล), prompt (ขยายข้อความให้โมเดล), หรือ local-jsx (แผง UI ด้วย Ink)

การค้นหา พิมพ์ / ใน REPL เพื่อดูคำสั่งทั้งหมดในบิลด์ของคุณ คำสั่งลงทะเบียนใน src/commands.ts และโหลดจากแหล่งในตัว, ปลั๊กอิน, สกิล, แบบรวม, และเวิร์กโฟลว์

การเติมคำอัตโนมัติแบบโต้ตอบ

Clew มีระบบข้อความ ghost ในตัวที่ทำนายอาร์กิวเมนต์ถัดไปขณะพิมพ์คำสั่ง:

  • พิมพ์ /goal ตามด้วยช่องว่าง ข้อความ ghost จะแสดง [condition] เป็นตัวอักษรสลัวหลังเคอร์เซอร์
  • กด Tab หรือ → (ลูกศรขวา) เพื่อยอมรับข้อความ ghost และเติม placeholder
  • รองรับคำแนะนำแบบต่อเนื่อง: /goal fix [name] [priority] เผยอาร์กิวเมนต์ที่เหลือเมื่อคุณเติมทีละอัน
  • คำสั่งที่มี argumentHint (เช่น /plugin-details <plugin-name>) แสดงคำแนะนำ placeholder แบบคงที่

สำหรับการเติมคำสั่งกลางการพิมพ์ ระบบข้อความ ghost เดียวกันเติมชื่อคำสั่งบางส่วน:

  • พิมพ์ /com ข้อความ ghost แสดง mit สำหรับ /commit
  • กด Tab หรือ → เพื่อเติมชื่อคำสั่ง

การโหลดคำสั่ง

คำสั่งถูกโหลดจากหลายแหล่งเมื่อเริ่มต้น:

  • ในตัว — คำสั่งหลักจาก src/commands/ (พร้อมใช้เสมอ)
  • สกิลที่มาพร้อม — สกิลที่มาพร้อมกับแอปพลิเคชัน
  • สกิลปลั๊กอินในตัว — จากปลั๊กอินในตัวที่เปิดใช้งาน
  • ไดเรกทอรีสกิล — ~/.clew/skills/ และ .clew/skills/ ของโปรเจกต์
  • คำสั่งปลั๊กอิน — จากปลั๊กอินที่ติดตั้ง
  • สกิลปลั๊กอิน — จากไดเรกทอรีสกิลของปลั๊กอิน
  • MCP skills — จากเซิร์ฟเวอร์ MCP (เมื่อเปิดใช้งาน MCP_SKILLS)
  • สกิลไดนามิก — ถูกค้นพบระหว่างการดำเนินการไฟล์
