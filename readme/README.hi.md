# Clew 🪽

Clew AI-सहायता प्राप्त सॉफ़्टवेयर विकास के लिए एक अनौपचारिक, अनुसंधान-उन्मुख CLI है।

यह एक स्रोत-निर्मित पुनर्निर्माण और विस्तार परियोजना है, जो स्थानीय विकास, डिबगिंग, स्व-होस्टेड वर्कफ़्लो और प्रदाता चयन के लिए डिज़ाइन की गई है।

यह रिपॉजिटरी कोई आधिकारिक उत्पाद, वितरण, भागीदार परियोजना या समर्थित कार्यान्वयन नहीं है।

> **अस्वीकरण:** यह परियोजना किसी तीसरे पक्ष से संबद्ध, समर्थित, प्रायोजित या अनुमोदित नहीं है। कृपया इस रिपॉजिटरी का उपयोग, संशोधन, पुनर्वितरण या परिनियोजन करने से पहले [LICENSE.md](../LICENSE.md) पढ़ें।

## यह परियोजना क्या प्रदान करती है

| क्षेत्र                 | विवरण                                                                                                               |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------- |
| स्रोत-निर्मित CLI       | एक Bun/TypeScript टर्मिनल एप्लिकेशन जिसे स्थानीय रूप से बनाया, परीक्षण किया, निरीक्षण किया और संशोधित किया जा सकता है |
| बहु-प्रदाता रूटिंग     | प्रदाता एडेप्टर और मॉडल चयन आदेशों के माध्यम से कई AI प्रदाताओं के लिए समर्थन                                          |
| डेवलपर टूलिंग          | संदर्भ निरीक्षण, कोड समीक्षा, सरलीकरण, शोध, प्लगिन, MCP, LSP, सत्र और पृष्ठभूमि वर्कफ़्लो के लिए आदेश                    |
| स्थानीय विस्तारशीलता    | प्लगिन, हुक, स्किल, कस्टम टूल, शेड्यूल किए गए कार्य और प्रोजेक्ट-स्तरीय कॉन्फ़िगरेशन के लिए समर्थन                     |
| अनुसंधान उपयोग          | AI कोडिंग एजेंट आर्किटेक्चर, टर्मिनल UX, प्रदाता रूटिंग और टूल निष्पादन के अध्ययन के लिए पारदर्शी कोडबेस                |

## विशेषताएँ

Clew सीधे आपके टर्मिनल में चलता है। यह स्थानीय कोडबेस का निरीक्षण और संपादन कर सकता है, अनुमतियों के साथ शेल कमांड निष्पादित कर सकता है, मॉडल प्रदाता बदल सकता है, और लंबे समय तक चलने वाले एजेंट वर्कफ़्लो को समन्वयित कर सकता है।

मुख्य विशेषताएँ:

* **बहु-प्रदाता AI रूटिंग** — Anthropic, OpenAI, Google Gemini, OpenRouter, Ollama, GitHub Copilot और OpenAI-संगत एंडपॉइंट का समर्थन
* **रनटाइम मॉडल स्विचिंग** — सत्र के दौरान मॉडल या प्रदाता बदलने के लिए `/model` का उपयोग करें
* **टूल-संचालित वर्कफ़्लो** — फ़ाइलें पढ़ें, खोजें, संपादित करें और लिखें; शेल कमांड निष्पादित करें; LSP क्वेरी करें; MCP टूल चलाएँ; ब्राउज़र ऑटोमेशन एकीकृत करें
* **प्लगिन हुक** — प्रॉम्प्ट, शेल निष्पादन, टूल कॉल, संदेश प्रदर्शन, सत्र प्रारंभ और फ़ाइल संपादन क्रियाओं में हुक
* **डायनामिक स्किल** — प्रोजेक्ट और `.claude/skills/` से स्किल लोड करें
* **कोड समीक्षा उपकरण** — परिवर्तनों की जाँच और लागू करने के लिए `/code-review --fix`, सफाई के लिए `/simplify` का उपयोग करें
* **गार्जियन ऑटो-रिव्यू** — `/guardian` सर्किट ब्रेकर के साथ LLM समीक्षक को अनुमति अनुरोध रूट करता है
* **PR प्रबंधन** — `/pr create`, `list`, `view`, `review`, `merge`, `status`
* **प्रदाता-स्वतंत्र रिमोट कंट्रोल** — WebSocket-आधारित CLI साझाकरण के लिए `/remote`
* **मॉडल पिकर** — वैश्विक या केवल-सत्र मॉडल चयन
* **प्लगिन मार्केटप्लेस** — प्लगिन स्रोतों के लिए `skipLfs` समर्थन
* **स्थानीय शोध** — कॉन्फ़िगर होने पर `/research <query>` के साथ स्थानीय वेब स्क्रेपिंग
* **एजेंट और पर्यवेक्षक** — पृष्ठभूमि एजेंट, बहु-चरणीय वर्कफ़्लो, सारांश, कार्य स्थिति, अनुमोदन और सत्र स्थिति प्रबंधित करें
* **पृष्ठभूमि शेल कमांड** — `!bg <command>` के साथ लंबे कमांड चलाएँ
* **शेड्यूल किए गए कार्य** — `/task` के साथ एक बार या आवर्ती कार्य बनाएँ
* **सत्र और ब्रिज मोड** — दूरस्थ वर्कफ़्लो के लिए सत्र सहेजें, पुनर्स्थापित करें और कनेक्ट करें

## त्वरित प्रारंभ

### वैश्विक स्थापना

```bash
npm install -g clew-code
```

या:

```bash
bun install -g clew-code
```

प्रोजेक्ट निर्देशिका में CLI चलाएँ:

```bash
clew
```

> वैश्विक लॉन्चर के लिए सिस्टम पर Bun स्थापित होना आवश्यक है

### स्रोत से चलाएँ

```bash
git clone https://github.com/ClewCode/ClewCode.git
cd ClewCode

bun install
bun run build
bun run start
```

डेवलपमेंट मोड:

```bash
bun run dev
```

## सिस्टम आवश्यकताएँ

- Bun 1.3 या उच्चतर
- Node.js 18 या उच्चतर
- Git
- Windows, macOS, Linux या WSL2
- कम से कम एक समर्थित प्रदाता से API कुंजी (Ollama जैसे स्थानीय प्रदाता का उपयोग करने पर आवश्यक नहीं)

## प्रदाता कॉन्फ़िगरेशन

शेल या `.env` फ़ाइल में प्रदाता कुंजियाँ सेट करें:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...
export GOOGLE_API_KEY=...
export OPENROUTER_API_KEY=sk-or-...
export OLLAMA_HOST=http://localhost:11434
```

सत्र के दौरान मॉडल/प्रदाता बदलें:

```text
/model
/model list
/model openai/gpt-4o
/model google/gemini-2.5-pro
```

प्रदाता दस्तावेज़:

```text
docs/providers.html
```

## शेड्यूल किए गए कार्य

शेड्यूल किए गए कार्य प्रणाली `/task` के माध्यम से उपलब्ध है।

```text
/task
```

उदाहरण:

```text
/task
Name: सर्वर जाँच
Schedule: Daily
Time: 20:00
Prompt: स्थानीय सर्वरों की स्थिति जाँचें
Storage: Durable
```

```text
/task
Name: कमिट रिमाइंडर
Schedule: In N minutes
Delay: 10
Prompt: मुझे कोड कमिट करने की याद दिलाएँ
Storage: Session-only
```

कार्य व्यवहार:

* Durable कार्य `.claude/scheduled_tasks.json` में सहेजे जाते हैं
* Session-only कार्य केवल सक्रिय सत्र के दौरान चलते हैं
* आवर्ती कार्य मानक 5-फ़ील्ड cron सिंटैक्स का उपयोग करते हैं
* एक बार के कार्य चलने के बाद हटा दिए जाते हैं
* शेड्यूल किए गए निष्पादन के लिए स्थानीय मशीन टाइमज़ोन का उपयोग किया जाता है

## Taste

Taste एक स्थानीय-प्रथम प्राथमिकता-सीखने का रनटाइम है। यह स्वीकार, अस्वीकार, संपादन, परीक्षण, लिंट और मैन्युअल नियमों से सीखता है। यह प्रतीकात्मक नियमों, शब्दार्थ प्राथमिकता स्कोरिंग और प्रासंगिक बैंडिट ऑप्टिमाइज़ेशन को जोड़ता है ताकि Clew को आपकी कोडिंग शैली के अनुकूल बना सके। यह बेस LLM को फ़ाइन-ट्यून नहीं करता।

```text
/taste                इंटरैक्टिव मेनू खोलें
/taste learn <rule>   मैन्युअल नियम जोड़ें
/taste forget <id>    नियम हटाएँ
/taste profile        सभी नियम दिखाएँ
/taste events         हाल के इवेंट दिखाएँ
/taste decay          विश्वास क्षय लागू करें
/taste eval           स्व-मूल्यांकन चलाएँ
/taste export         उच्च-विश्वास वाले नियम निर्यात करें
/taste import <file>  फ़ाइल से नियम आयात करें
/taste on             Taste सक्षम करें
/taste off            Taste अक्षम करें
```

### मुख्य क्षमताएँ

- **इंटरैक्टिव मेनू** — तीर-कुंजी नेविगेबल डायलॉग जिसमें 11 क्रियाएँ हैं, एसिंक्रोनस संचालन के लिए Spinner लोडिंग
- **संपादन सत्यापन** — अनुमति अनुरोध के दौरान संपादन स्कैन करता है, सीखे गए नियमों के उल्लंघन पर चेतावनी देता है
- **कॉन्फ़िग लाइव-रिलोड** — `subscribeToSettingsChanges()` के माध्यम से `settings.json` परिवर्तनों की सदस्यता लेता है
- **स्थिति पंक्ति** — PromptInputFooter में `ⓘ taste: N rules` दिखाया गया
- **प्रॉम्प्ट इंजेक्शन** — सिस्टम प्रॉम्प्ट में 8 प्रासंगिक नियमों तक के साथ `<clew_taste>` XML ब्लॉक इंजेक्ट करता है
- **सिग्नल संग्रह** — PermissionContext और टूल निष्पादन से फायर-एंड-फ़ॉरगेट सिग्नल
- **क्षय इंजन** — अप्रयुक्त नियमों के लिए क्रमिक विश्वास कमी (आधे-जीवन आधारित, डिफ़ॉल्ट 30 दिन)

पूर्ण दस्तावेज़ के लिए [docs/taste.html](../docs/taste.html) देखें।

## डेवलपमेंट

```bash
bun run dev              # डेवलपमेंट मोड शुरू करें
bun run start            # स्रोत से CLI चलाएँ
bun run build            # dist/ में बनाएँ
bun test                 # परीक्षण चलाएँ
bun x tsc --noEmit       # टाइप जाँच
bun run lint:check       # Biome lint नियम जाँचें
bun run format:check     # Biome फ़ॉर्मेटिंग जाँचें
bun run check:ci         # Biome CI सत्यापन चलाएँ
```

डेवलपर उपयोगिताएँ:

```bash
bun run preload <module>     # मॉड्यूल संदर्भ प्रीलोड करें
bun run session <command>    # सत्र संदर्भ सहेजें, सूचीबद्ध करें या पुनर्स्थापित करें
bun run codegraph            # मॉड्यूल निर्भरता ग्राफ़ उत्पन्न करें
bun run ast-grep -- <args>   # संरचनात्मक AST खोज या पुनर्लेखन चलाएँ
```

src/

├── agentRuntime/         # एजेंट ऑर्केस्ट्रेशन और स्थायी रन स्टोर
├── commands/             # स्लैश कमांड कार्यान्वयन
├── tools/                # अंतर्निहित डेवलपर उपकरण
├── services/
│   ├── ai/               # प्रदाता प्रबंधक, एडेप्टर, सामान्यीकरण और providers.json
│   ├── mcp/              # Model Context Protocol क्लाइंट
│   ├── plugins/          # प्लगिन जीवनचक्र हुक और इंटरसेप्टर

│   └── SessionMemory/    # स्थायी सत्र स्मृति
├── skills/               # गतिशील स्किल लोडर

└── vim/                  # Vim-जैसा नेविगेशन मोड

## आर्किटेक्चर

```text
Terminal UI
  -> कमांड रजिस्ट्री और कीबाइंडिंग
  -> प्रदाता प्रबंधक और AI एडेप्टर
  -> क्वेरी इंजन और स्ट्रीमिंग लूप
  -> टूल निष्पादक सेवा
  -> प्लगिन, MCP, LSP, एजेंट, सत्र स्मृति और ब्रिज
```

## प्लेटफ़ॉर्म नोट्स

### Windows

```powershell
Remove-Item -Recurse -Force node_modules
bun install
bun run dev
```

Windows के लिए एक प्रीकंपाइल्ड `ripgrep` बाइनरी इस पथ पर बंडल हो सकती है:

```text
src/utils/vendor/ripgrep/x64-win32/rg.exe
```
