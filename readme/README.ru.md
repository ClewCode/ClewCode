# Clew 🪽

Clew — неофициальный, ориентированный на исследования CLI для разработки программного обеспечения с помощью ИИ.

Этот проект представляет собой реконструкцию и расширение исходного кода, предназначенную для локальной разработки, отладки, самостоятельных рабочих процессов и свободы выбора провайдера.

| Маршрутизация провайдеров | Поддержка нескольких ИИ-провайдеров через адаптеры и команды выбора модели                                             |

## Возможности

Clew работает прямо в вашем терминале. Он может проверять и редактировать локальные кодовые базы, выполнять команды shell с разрешениями, переключать провайдеров моделей и координировать длительные рабочие процессы агентов.

Ключевые возможности:

* **Маршрутизация между провайдерами ИИ** — Поддерживает Anthropic, OpenAI, Google Gemini, OpenRouter, Ollama, GitHub Copilot и совместимые с OpenAI endpoints
* **Переключение модели во время выполнения** — Используйте `/model` для переключения модели или провайдера во время сессии
* **Рабочие процессы на основе инструментов** — Читайте, ищите, редактируйте и записывайте файлы; выполняйте команды shell; запрашивайте LSP; запускайте инструменты MCP; интегрируйте автоматизацию браузера
* **Хуки плагинов** — Подключайтесь к промптам, выполнению shell, вызовам инструментов, отображению сообщений, запуску сессии и действиям по редактированию файлов
* **Динамические навыки** — Загружайте навыки из проекта и `.claude/skills/`
* **Инструменты ревью кода** — Используйте `/code-review --fix` для проверки и применения изменений, `/simplify` для очистки кода
* **Автоматическое ревью Guardian** — `/guardian` направляет запросы разрешений рецензенту LLM с автоматическим выключателем
* **Управление PR** — `/pr create`, `list`, `view`, `review`, `merge`, `status`
* **Независимое от провайдера удаленное управление** — `/remote` для обмена CLI через WebSocket
* **Выбор модели** — Глобальный выбор модели или только для сессии
* **Маркетплейс плагинов** — Поддержка `skipLfs` для источников плагинов
* **Локальное исследование** — `/research <query>` для исследования с локальным веб-скрапингом
* **Агенты и супервизор** — Управление фоновыми агентами, многошаговыми процессами, сводками, статусом задач, утверждениями и состоянием сессии
* **Фоновые команды shell** — Выполняйте длительные команды с `!bg <command>`
* **Запланированные задачи** — Создавайте одноразовые или повторяющиеся задачи с `/task`
* **Сессии и bridge-режим** — Сохраняйте, восстанавливайте и подключайте сессии для удаленных рабочих процессов

## Быстрый старт

### Глобальная установка

```bash
npm install -g clew-code
```

Или:

```bash
bun install -g clew-code
```

Запустите CLI в директории проекта:

```bash
clew
```

> Глобальный лаунчер требует установки Bun в системе

### Запуск из исходников

```bash
git clone https://github.com/ClewCode/ClewCode.git
cd ClewCode

bun install
bun run build
bun run start
```

Режим разработки:

```bash
bun run dev
```

## Системные требования

- Bun 1.3 или выше
- Node.js 18 или выше
- Git
- Windows, macOS, Linux или WSL2
- API-ключ хотя бы от одного поддерживаемого провайдера (не требуется при использовании локального провайдера, например Ollama)

## Настройка провайдеров

Установите ключи провайдеров в shell или файле `.env`:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...
export GOOGLE_API_KEY=...
export OPENROUTER_API_KEY=sk-or-...
export OLLAMA_HOST=http://localhost:11434
```

Переключение модели/провайдера во время сессии:

```text
/model
/model list
/model openai/gpt-4o
/model google/gemini-2.5-pro
```

Документация провайдеров:

```text
docs/providers.html
```

## Запланированные задачи

Система запланированных задач доступна через `/task`.

```text
/task
```

Примеры:

```text
/task
Name: Проверка сервера
Schedule: Daily
Time: 20:00
Prompt: Проверить статус локальных серверов
Storage: Durable
```

```text
/task
Name: Напоминание о коммите
Schedule: In N minutes
Delay: 10
Prompt: Напомнить мне сделать коммит
Storage: Session-only
```

Поведение задач:

* Постоянные задачи сохраняются в `.claude/scheduled_tasks.json`
* Задачи только для сессии выполняются только во время активной сессии
* Повторяющиеся задачи используют стандартный 5-польный синтаксис cron
* Одноразовые задачи удаляются после выполнения
* Для запланированного выполнения используется местный часовой пояс

## Taste

Taste — это локальная среда обучения предпочтениям. Она учится на сигналах принятия, отклонения, редактирования, тестирования, линтинга и ручных правил. Она объединяет символьные правила, семантическую оценку предпочтений и контекстную бандитскую оптимизацию для адаптации Clew к вашему стилю кодирования. Она не выполняет точную настройку базовой LLM.

```text
/taste                Открыть интерактивное меню
/taste learn <rule>   Добавить ручное правило
/taste forget <id>    Удалить правило
/taste profile        Показать все правила
/taste events         Показать последние события
/taste decay          Применить снижение доверия
/taste eval           Запустить самооценку
/taste export         Экспортировать правила с высоким доверием
/taste import <file>  Импортировать правила из файла
/taste on             Включить Taste
/taste off            Выключить Taste
```

### Ключевые возможности

- **Интерактивное меню** — Диалог с навигацией стрелками и 11 действиями, Spinner для асинхронных операций
- **Проверка редактирования** — Сканирует изменения во время запроса разрешений, предупреждает о нарушениях изученных правил
- **Горячая перезагрузка конфигурации** — Подписывается на изменения `settings.json` через `subscribeToSettingsChanges()`
- **Строка состояния** — `ⓘ taste: N rules` отображается в PromptInputFooter
- **Внедрение в промпт** — Внедряет XML-блок `<clew_taste>` с до 8 релевантными правилами в системный промпт
- **Сбор сигналов** — Сигналы fire-and-forget из PermissionContext и выполнения инструментов
- **Двигатель снижения** — Постепенное снижение доверия к неиспользуемым правилам (на основе периода полураспада, по умолчанию 30 дней)

Полная документация в [docs/taste.html](../docs/taste.html).

## Разработка

```bash
bun run dev              # Запустить режим разработки
bun run start            # Запустить CLI из исходников
bun run build            # Собрать в dist/
bun test                 # Запустить тесты
bun x tsc --noEmit       # Проверка типов

bun run format:check     # Проверить форматирование Biome
bun run check:ci         # Запустить валидацию Biome CI
```

Утилиты разработки:

```bash
bun run preload <module>     # Предзагрузить контекст модуля
bun run session <command>    # Сохранить, вывести список или восстановить контекст сессии
bun run codegraph            # Создать графы зависимостей модулей
bun run ast-grep -- <args>   # Выполнить структурный поиск или перезапись AST
```

## Структура проекта

```text
src/
├── main.tsx              # Загрузка UI терминала и основной цикл
├── query.ts              # Обработка запросов и логика системного промпта
├── QueryEngine.ts        # Оркестрация запросов, кэширование, дедупликация и ограничение скорости
├── agentRuntime/         # Оркестрация агентов и постоянные хранилища выполнения
├── commands/             # Реализации слэш-команд
├── tools/                # Встроенные инструменты разработчика
├── services/
│   ├── ai/               # Менеджер провайдеров, адаптеры, нормализаторы и providers.json
│   ├── mcp/              # Клиенты Model Context Protocol
│   ├── plugins/          # Хуки и перехватчики жизненного цикла плагинов
│   ├── tools/            # Сервис выполнения инструментов
│   ├── lsp/              # Интеграция Language Server Protocol
│   ├── Supervisor/       # Супервизор фоновых агентов
│   └── SessionMemory/    # Постоянная память сессий
├── skills/               # Динамический загрузчик навыков
├── cli/                  # Контексты UI терминала
├── components/           # Компоненты UI терминала
├── bridge/               # WebSocket-мост
├── coordinator/          # Многоагентный координатор
├── keybindings/          # Сочетания клавиш
├── state/                # Реактивные хранилища
└── vim/                  # Режим навигации как в Vim
```

## Архитектура

```text
Terminal UI
  -> Реестр команд и сочетания клавиш
  -> Менеджер провайдеров и ИИ-адаптеры
  -> Двигатель запросов и циклы потоковой передачи
  -> Сервис выполнения инструментов
  -> Плагины, MCP, LSP, агенты, память сессий и мост
```
